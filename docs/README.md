# Documentação do Projeto

Bem-vindo à documentação do projeto! Este diretório foi organizado para ajudar desenvolvedores (especialmente iniciantes) a entenderem o ciclo de vida, a arquitetura e os padrões que seguimos.

## Estrutura de Diretórios

### [01 - Processo de Desenvolvimento](./01-processo/)
Contém o passo a passo de como transformamos uma ideia em um produto final.
- **1. Ideia:** Definição inicial e objetivos.
- **2. Pesquisa:** Levantamento de requisitos e referências.
- **3. Protótipo:** Design inicial e validação de conceito.
- **4. PRD (Product Requirements Document):** Especificações detalhadas do produto.
- **5. Kanban:** Organização das tarefas e fluxo de trabalho.
- **6. Execução:** Melhores práticas durante o desenvolvimento.
- **7. QA (Garantia de Qualidade):** Processos de teste e validação.

### [02 - Arquitetura](./02-arquitetura/)
Documentação técnica sobre a estrutura do sistema.
- **Análise de Ponto de Função:** Medição de tamanho e complexidade.
- **Segurança API:** Padrões e protocolos de segurança utilizados.

### [03 - Guias e Configuração](./03-guias-e-configuracao/)
Instruções para configurar o ambiente e manter a consistência do código.
- **Setup LLM Local:** Como configurar modelos de linguagem localmente.
- **Migração:** Procedimentos para migração de sistemas ou dados.
- **Estilo e Padrões:** Guia de estilo de código e convenções do projeto.

### [04 - Conhecimento Java](./04-conhecimento-java/)
Recursos específicos para o ecossistema Java.
- **Notas de Estudo:** Insights e aprendizados sobre Java.
- **Vídeos e Tutoriais:** Referências visuais para aprendizado.

### [05 - Artigos](./05-artigos/)
Conteúdos editoriais e artigos relacionados ao projeto ou tecnologias utilizadas.

### [Assets](./assets/)
Imagens, diagramas e recursos visuais utilizados na documentação.

---

## Como contribuir
Para adicionar nova documentação, escolha a categoria mais adequada e utilize arquivos Markdown. Se for adicionar imagens, coloque-as na pasta `assets`.
