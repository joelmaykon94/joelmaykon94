É tão simples quanto isso. Não quero dizer apenas juntar um código que faça o que você precisa agora, mas sim escrever um código de qualidade de produção que seja limpo, legível, testável e implantável. Assim, a engenharia de software é o design, desenvolvimento, implantação e manutenção de aplicações de software. E não faz muito tempo, não havia tal coisa como engenharia de dados como uma profissão oficial. Havia apenas engenheiros de software que ocasionalmente lidavam com dados em seu trabalho. Eventualmente, à medida que as empresas reconheceram o valor dos dados, os engenheiros de software começaram a assumir vários aspectos da engenharia de dados como parte de seu trabalho. Com a crescente diversidade e volume de dados nas últimas décadas, o componente orientado a dados da engenharia de software se tornou muito mais intensivo e, eventualmente, emergiu como um campo próprio.

Ao longo dos anos, esses engenheiros de software que faziam engenharia de dados construíram uma variedade de soluções fantásticas, de modo que hoje, como engenheiro de dados, você tem acesso a uma ampla gama de serviços gerenciados e aplicações que permitem que você faça o trabalho de forma mais eficiente. E isso é algo bom. Isso permite que você passe mais tempo focado nos aspectos mais importantes de agregar valor real à sua organização. No entanto, isso não significa que a codificação não seja importante no seu trabalho como engenheiro de dados. Na verdade, é mais importante do que nunca que você consiga escrever um código excelente e que o código que você escrever seja de alta qualidade.
