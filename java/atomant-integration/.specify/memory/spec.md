# Technical Specification: `atomant-integration` (Integration Module)

This document specifies the technical design, OpenAPI contracts, and Java NIO implementations for exporting accounting files and receiving reconciliation webhooks.

---

## 1. OpenAPI 3.0 Contract (Swagger)

Exposes endpoints to generate CSV accounting reports and handle reconciliation callback webhooks.

```yaml
openapi: 3.0.3
info:
  title: Integration Module API
  version: 1.0.0
  description: Technical specification for outbound accounting file exports and inbound reconciliation callback webhooks.
paths:
  /api/v1/integration/export/accounting:
    get:
      summary: Export Standardized Accounting File
      description: Generates a CSV file containing aggregated daily branch/agency fees for external accounting systems.
      responses:
        '200':
          description: Standardized CSV report file returned.
          headers:
            Content-Disposition:
              schema:
                type: string
                example: "attachment; filename=\"accounting-fees-2026-06-06.csv\""
          content:
            text/csv:
              schema:
                type: string
                format: binary

  /api/v1/integration/reconciliation:
    post:
      summary: Receive Reconciliation Webhook
      description: Receives asynchronous callback status from external accounting/ledger systems to confirm batch ingestion.
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/ReconciliationCallback'
      responses:
        '202':
          description: Webhook received and scheduled for reconciliation.

components:
  schemas:
    ReconciliationCallback:
      type: object
      properties:
        batchId:
          type: string
          format: uuid
        status:
          type: string
          enum: [ACCEPTED, REJECTED]
        message:
          type: string
```

## 3. Java NIO File Generation Service (`AccountingExportService.java`)

Demonstrates how to write CSV text reports using Java NIO `BufferedWriter` and path structures to handle memory safely.

```java
package org.acme.integration.domain.service;

import jakarta.enterprise.context.ApplicationScoped;
import java.io.BufferedWriter;
import java.io.IOException;
import java.math.BigDecimal;
import java.nio.charset.StandardCharsets;
import java.nio.file.Files;
import java.nio.file.Path;
import java.time.LocalDate;
import java.util.List;

@ApplicationScoped
public class AccountingExportService {

    public Path generateAccountingCsv(LocalDate date, List<AccountingRecordMock> records) throws IOException {
        // Create a temporary file in the OS temp directory using Java NIO
        Path tempFile = Files.createTempFile("accounting-fees-", ".csv");

        // Write content line-by-line using buffered NIO writer
        try (BufferedWriter writer = Files.newBufferedWriter(tempFile, StandardCharsets.UTF_8)) {
            // Write CSV Header
            writer.write("branch_code,calculation_date,total_fee_cents");
            writer.newLine();

            // Write CSV Records
            for (AccountingRecordMock record : records) {
                String line = String.format("%s,%s,%d",
                        record.branchCode(),
                        record.date().toString(),
                        record.totalFeeCents().longValue());
                writer.write(line);
                writer.newLine();
            }
        }

        return tempFile;
    }

    // Mock record representing inner data mapping
    public record AccountingRecordMock(
        String branchCode,
        LocalDate date,
        BigDecimal totalFeeCents
    ) {}
}
```

## 4. Quarkus REST Controller Endpoint (`IntegrationController.java`)

Implements JAX-RS methods using Java 21 Virtual Threads (`@RunOnVirtualThread`). It reads the temporary file generated by NIO and streams it back to the REST client before deleting it.

```java
package org.acme.integration.api;

import io.smallrye.common.annotation.RunOnVirtualThread;
import jakarta.inject.Inject;
import jakarta.ws.rs.*;
import jakarta.ws.rs.core.MediaType;
import jakarta.ws.rs.core.Response;
import org.acme.integration.domain.model.ReconciliationCallback;
import org.acme.integration.domain.service.AccountingExportService;
import java.io.IOException;
import java.math.BigDecimal;
import java.nio.file.Files;
import java.nio.file.Path;
import java.time.LocalDate;
import java.util.List;

@Path("/api/v1/integration")
@Produces(MediaType.APPLICATION_JSON)
@Consumes(MediaType.APPLICATION_JSON)
public class IntegrationController {

    @Inject
    AccountingExportService exportService;

    @GET
    @Path("/export/accounting")
    @Produces("text/csv")
    @RunOnVirtualThread
    public Response exportAccountingFile() {
        try {
            LocalDate date = LocalDate.now();
            
            // Mock database results for CSV representation
            List<AccountingExportService.AccountingRecordMock> mockDbData = List.of(
                    new AccountingExportService.AccountingRecordMock("0001", date, BigDecimal.valueOf(150450)),
                    new AccountingExportService.AccountingRecordMock("0002", date, BigDecimal.valueOf(340980))
            );

            // Generate file using Java NIO
            Path csvFilePath = exportService.generateAccountingCsv(date, mockDbData);
            byte[] fileBytes = Files.readAllBytes(csvFilePath);
            
            // Clean up temp file immediately after reading
            Files.deleteIfExists(csvFilePath);

            return Response.ok(fileBytes)
                    .header("Content-Disposition", "attachment; filename=\"accounting-fees-" + date + ".csv\"")
                    .build();
            
        } catch (IOException e) {
            return Response.status(Response.Status.INTERNAL_SERVER_ERROR)
                    .entity("Error generating accounting report: " + e.getMessage())
                    .type(MediaType.TEXT_PLAIN)
                    .build();
        }
    }

    @POST
    @Path("/reconciliation")
    @RunOnVirtualThread
    public Response receiveReconciliationCallback(ReconciliationCallback callback) {
        if (callback == null || callback.batchId() == null || callback.status() == null) {
            return Response.status(Response.Status.BAD_REQUEST).entity("Invalid callback body").build();
        }

        // Process reconciliation log status (e.g. update Audit DB entry state)
        System.out.println("Processing Batch Reconciliation: ID " + callback.batchId() + " Status: " + callback.status());

        return Response.status(Response.Status.ACCEPTED).build();
    }
}
```
